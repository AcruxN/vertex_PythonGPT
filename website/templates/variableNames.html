Variable Names

Variable Naming

A variable may feature a concise designation (such as x and y) or a more elaborate identifier (e.g., age, car_name, total_volume). Guidelines for Python variables:
  • The name of a variable must start with a letter or an underscore character
  • A variable's name cannot start with a number
  • A variable's name can only contain alphanumeric characters and underscores (A-z, 0-9, and _ )
  • Variable names are case-sensitive (age, Age, and AGE are different variables)
  • A variable's name cannot be one of Python's reserved keywords.
Example
Legal variable names:
myvar = "Ian"
my_var = "Ian"
_my_var = "Ian"
myVar = "Ian"
MYVAR = "Ian"
myvar2 = "Ian"
Example
Illegal variable names:
2myvar = "Ian"
my-var = "Ian"
my var = "Ian"
Note: Remember that variable names are case-sensitive
Multi-word Variable Names

Variable names containing multiple words can pose readability challenges.

There are several approaches you can employ to enhance their readability:

Camel Case

Each word, except the first, begins with a capital letter:

myVariableName = "Ian"
Pascal Case

Each word starts with a capital letter:

MyVariableName = "Ian"
Snake Case

Each word is delimited by an underscore character:

my_variable_name = "Ian"
